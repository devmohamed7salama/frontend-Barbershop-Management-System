# 💈 نظام إدارة صالون المقص الذهبي (الواجهة الأمامية - Frontend)

واجهة مستخدم حديثة وفاخرة مصممة خصيصاً لإدارة صالونات الحلاقة الراقية وحجز المواعيد وإصدار الفواتير. تم بناء هذا المشروع باستخدام **React** و**Vite** لتقديم تجربة مستخدم سريعة وسلسة.

---

## 🛠️ التقنيات المستخدمة (Tech Stack)

1. **Vite + React (HMR)**: بيئة عمل فائقة السرعة مع تحديث فوري أثناء التطوير.
2. **React Router Dom (v6)**: لإدارة التنقل والمسارات والتحويل التلقائي للمسارات المحمية.
3. **React Bootstrap**: لتوفير استجابة كاملة وتصميم متناسق وسريع (Responsive Layout).
4. **Vanilla CSS (App.css & index.css)**: تخصيص وتجميل الواجهات، مع تدرجات لونية، ومؤثرات تحوم (Hover)، وحركات انتقالية سلسة تعكس الطابع الفاخر للصالون (الذهبي والأسود والأبيض المريح).
5. **Axios**: لإرسال الطلبات إلى الباك أند (Laravel API)، مع معترضات (Interceptors) لإرفاق توكن المصادقة تلقائياً.
6. **React Toastify**: لعرض التنبيهات المنبثقة التفاعلية بنجاح أو فشل العمليات.
7. **Google Fonts (Cairo)**: الخط الأساسي للمشروع لإضفاء لمسة جمالية عربية احترافية.
8. **Material Symbols Outlined**: لتوفير أيقونات حديثة وعالية الجودة في جميع القوائم والصفحات.

## 📁 هيكلية المجلدات (Folder Structure)

```text
src/
├── api/          # إعداد عميل Axios ومصادقة المستخدم
├── assets/       # الصور والملفات الثابتة
├── components/   # المكونات المشتركة ونماذج التخطيط (Layouts) وحماية المسارات
├── context/      # سياق المصادقة (AuthContext) لمتابعة الجلسات
├── hooks/        # الخطافات المخصصة (Custom Hooks) مثل useApi مع ذاكرة كاش و useDebounce
├── pages/        # صفحات النظام (العامة والخاصة بالإدارة)
├── routes/       # تعريف المسارات والربط بين المكونات والصفحات
├── services/     # وظائف API المقسمة حسب النماذج (المواعيد، الخدمات، الفواتير، الحلاقين)
├── App.css       # الأنماط والتصميم البصري العام للفرونت أند
├── App.jsx       # نقطة الانطلاق الأساسية للتطبيق وإعداد موفر الجلسة والتنبيهات
└── main.jsx      # تفعيل التطبيق وربطه بـ DOM
```

## 🌐 تفصيل الصفحات (App Pages)

ينقسم النظام إلى مسارين رئيسيين: **صفحات عامة للعملاء والزوار**، و**لوحة تحكم خاصة بالإدارة (Admin Panel)**.

### أولاً: مسارات العملاء والزوار (Client Layout)
* **الصفحة الرئيسية وحجز الموعد (`/` أو `/book`)**:
  - شاشة ترحيبية فاخرة تدعو العميل للحجز الفوري.
  - عرض تفاعلي للخدمات المقدمة من الصالون مع تفاصيل الأسعار وفترات الخدمات.
  - عرض الحلاقين المتميزين بالصالون (يظهر القسم ديناميكياً إذا كان عدد الحلاقين بالصالون أكبر من 2).
  - نموذج حجز ذكي يحسب وقت الحجز التقديري والسعر الكلي مباشرة، ويمنع اختيار مواعيد قديمة، ويدعم تسجيل البيانات للزائرين (حجز سريع) أو الحسابات المسجلة تلقائياً.
* **صفحة حجوزاتي (`/my-appointments`)**:
  - تظهر للعملاء المسجلين لمتابعة حالة حجوزاتهم (الحجوزات القادمة القائمة، والحجوزات السابقة المكتملة أو الملغاة).
* **صفحات الدخول والتسجيل (`/login` & `/register`)**:
  - تمكين المستخدمين من إنشاء حسابات برقم الهاتف والاسم لربط حجوزاتهم التاريخية تلقائياً بحسابهم الجديد.

### ثانياً: لوحة تحكم الإدارة (Admin Dashboard Layout)
* **الرئيسية والإحصائيات (`/dashboard`)**: نظرة سريعة على المبيعات اليومية، عدد الحلاقين، الخدمات، المبيعات النشطة، والشيفت الحالي.
* **الشيفتات (`/shifts`)**: إدارة فتح وإغلاق شيفتات العمل لضبط الخزينة وحساب الإيرادات اليومية.
* **المواعيد (`/appointments`)**: استعراض كافة حجوزات الصالون، قبولها، إنهائها، أو إلغائها مع إشعار العملاء.
* **الحلاقين (`/barbers`)**: إدارة موظفي الصالون (إضافة حلاق جديد، تعديل بياناته، إدارته).
* **العملاء (`/customers`)**: استعراض كافة العملاء المسجلين بالصالون وتاريخ زياراتهم.
* **الخدمات (`/services`)**: التحكم بقائمة الخدمات المتاحة بالصالون، أسعارها، فتراتها الزمنية، وحالتها (نشطة/غير نشطة).
* **الفواتير (`/invoices`)**: 
  - استعراض كافة الفواتير الصادرة بالصالون مع تصفية حسب التاريخ.
  - **إنشاء فاتورة جديدة (`/invoices/new`)**: إرفاق فاتورة بناءً على موعد حجز مؤكد وعملاء حقيقيين.
  - **فاتورة سريعة (`/invoices/quick`)**: إصدار فاتورة مباشرة للعملاء الزائرين دون الحاجة لحجز مسبق.
  - **عرض وتفاصيل الفاتورة**: مودال يحتوي على تفاصيل الفاتورة كاملة والخدمات المقدمة، وتوفر أزرار إجراءات جديدة للطباعة، العرض، والإرسال عبر واتساب برقم العميل.

## 🚀 كيفية التشغيل والتهيئة المحلية (Installation & Setup)

1. **تثبيت الملحقات**:
   ```bash
   npm install
   ```

2. **ملف الإعدادات البيئية (`.env` أو `.env.local`)**:
   تأكد من إعداد رابط الباك أند الصحيح (API URL):
   ```env
   VITE_API_URL=http://localhost:8000/api
   ```

3. **تشغيل سيرفر التطوير المحلي**:
   ```bash
   npm run dev
   ```

4. **بناء نسخة الإنتاج**:
   ```bash
   npm run build
   ```
